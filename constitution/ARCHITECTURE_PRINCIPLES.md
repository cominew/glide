# Glide OS Architecture Principles
## 架构原则

---

## 1. Event Native

Everything is event-driven.

No component owns reality.

---

## 2. No Global Mind

No centralized intelligence layer exists.

Hooks, stores, or controllers must never
simulate a system consciousness.

---

## 3. Projection UI

UI subscribes to events.

UI never derives truth.

---

## 4. Cognition / Execution Separation

Thinking output:
- observable
- non-executable

Skill output:
- executable
- authoritative

---

## 5. Observation Instead of State

State containers are temporary observation buffers.

They must be replaceable without consequence.

---

## 6. Scheduler Is Not Consciousness

Schedulers do not awaken systems.

They only sequence events.

Heartbeat loops are discouraged.

---

## 7. Replaceability

Any module must be removable
without metaphysical damage to Glide.

可替代性是最高工程原则。
